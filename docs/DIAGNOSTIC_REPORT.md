# DIAGNOSTIC REPORT: 应对医学图像生成中感知-失真权衡及 VAE 伪影的实证分析

## 1. 实验回顾与核心问题
目前我们使用 OmniGen 生成多角度胸部 X 光（CXR）图像时，出现了显著的“指标倒挂”现象：
- **失真类指标极高**：结构相似度（SSIM: ~0.65）极好。
- **感知类指标极低**：感知距离（LPIPS: ~0.188）和特征分布距离（FID 偏高）表现糟糕。
- **初始猜想**：我们曾推测，这是由于 SDXL VAE 在自然图像预训练后处理纯灰度图像产生的 **全局亮度/对比度漂移**。这种漂移不会严重破坏依靠局部结构计算的 SSIM，但会毁掉基于深度特征提取的 FID 和 LPIPS。

为此，我们在包含 `cxr_finetune_lora_8000` 预测结果的数据集上进行了 **1000个样本** 的后处理诊断实验，试图用包含 GT 信息的 **直方图匹配 (Histogram Matching)** 来强行纠正亮度/对比度漂移，并观察指标变化。

## 2. 实验数据（1000样本评估）
使用 `diagnose_metrics.py` 执行直方图匹配并在内存中重新计算指标，结果如下：

| Metric | Original Pred (基准) | Matched Pred (匹配后) | 变化趋势 |
|---|---|---|---|
| **SSIM** | 0.6545 | 0.6417 | ⬇️ 微降 |
| **PSNR** | 19.79 | 20.61 | ⬆️ 提升 |
| **LPIPS**| 0.1886 | 0.1865 | ➡️ 几乎不变 |
| **FID** | 51.50 | 54.78 | ⬇️ 恶化 |

## 3. 诊断分析：推翻旧猜想，发现真问题
实验数据给出了一个明确而冷酷的结论：**全局亮度/对比度域漂移决不是导致 FID/LPIPS 崩塌的核心元凶。**

如果仅仅是全局映射问题，经过精准的像素重分布后，感知指标应该大幅回血。但感知指标纹丝不动，FID 甚至变得**更差**。这在学术上引出了极其深刻的诊断推论：

### 3.1 VAE 产生的不是宏观映射偏差，而是“微观高频伪影”
SSIM 极高说明模型在解剖结构（肋骨、心脏轮廓）等**大尺度拓扑**上对齐非常完美（归功于 OmniGen 优秀的 Cross-Attention 机制）。
但底层的 VAE 在解码医学单通道图像时，引入了肉眼容易忽略，但被 Inception 和 AlexNet 极其痛恨的 **高频结构性噪点**（如 Checkerboard Artifacts 棋盘格伪影、边缘高频抖动、不自然的类织物质感）。直方图属于点运算映射，无法消除这些微观几何伪影。

### 3.2 为什么直方图匹配反而使得 FID 恶化、SSIM 下降？
当你强行把带有“微观高频伪影”的预测图去拟合高对比度的真实 X 光直方图时，这种非线性的像素拉伸**直接放大了**原本在暗部或亮部由于对比度低而不明显的高频噪点。
- **FID 更差**：InceptionV3 捕捉到了这些被对比度拉伸放大的、对医学图像来说完全病态的奇异高频纹理。
- **SSIM 微降**：局部的高频噪点被显著放大后，彻底恶化了局部高斯窗口内的协方差平滑性，导致 SSIM 下滑。
